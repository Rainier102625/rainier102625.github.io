## 1. 개요

### KDT 2차 프로젝트 및 공모전 참가

* 디지털 사회혁신 서비스 개발 공모전을 참가를 주요 방향성으로 제시
* 팀원 중에 이전에 관심이 있던 역사마스터 API를 이용한 아이디어로 시작
* 역사마스터 API = 서울시 지하철 역 또는 시내버스 정류장 주위 반경으로 여행지를 보여주는 API
* 여기에 혼잡도를 추가한 혼잡한 여행지를 고려할 때 비슷한 여행지로 우회추천하는 형식에 지도를 만들자
* 이를 kakao 지도맵 API와 연동해서 웹 완성을 목표로 함

### 데이터 센터 화재 사건 발발

* 프로젝트 진행중 데이터센터 화재로 서울시 실시간 도시데이터 사용 불가  

#### 트러블 슈팅

1. 사이트에 유동인구 7년치 학습데이터가 있는걸 전부 수집해 현재 시간을 예측하는 API 서버 생성
2. 정확한 위치 정보를 위해 집계구 단위를 범주형 변수로 지정
3. 이를 csv파일에 feature를 맞춰서 데이터 가공 후 parquet 포맷으로 변경
4. lgbm 모델이 이 csv 데이터를 전부 받아들이지 못해서 코로나기간을 제외한 나머지 기간의 데이터를 1년치로 나눔
5. 학습들을 1년치씩 나눈것을 다시 합한 후 유동인구 예측
6. 이를 SK 장소혼잡도 공식으로 계산 후 Flask서버로 배포하여 Spring boot 서버에서 받아서 그 현 위치에 혼잡도를 알려줌
