## 1. 개요

### 요약

본 프로젝트는 여가 및 문화 생활의 폭발적 증가에 따라 특정 시기·장소에 집중되는 인파로 인한 혼잡, 이동 지연, 현장 과밀 등의 사회적 문제를 해결하기 위해 기획된 지능형 혼잡도 지도 및 관광 추천 서비스입니다.  
특별한 행사와 축제 등에서의 군중 붐빔은 안전사고 위험까지 높이고 있는데, 실제 경찰·지자체도 실시간 데이터 기반 사고 예방 시스템의 도입을 강력히 요구하고 있습니다.  
근거자료([1](https://www.yna.co.kr/view/AKR20241025078600004), [2](https://news.nate.com/view/20251028n39852))  
이 서비스는 혼잡도 데이터를 AI로 예측·시각화하고, 챗봇 기반 추천을 통해 각 사용자별로 혼잡도가 낮은 최적의 경로나 대체 장소를  제시함으로써, 과밀지역의 인파 분산과 사용자 개별 만족도를 동시에 실현합니다.  
NHN 클라우드의 K-PaaS 플랫폼 위에 서비스를 배포하고, CI/CD 파이프라인을 통해 소스 코드의 자동 빌드·배포·테스트를 수행하여 클라우드 네이티브 환경에서 안정적으로 운영됩니다.  

### KDT 2차 프로젝트 및 공모전 참가

* 개방형 클라우드 플랫폼(K-PaaS) 활용 공모전을 참가를 주요 방향성으로 제시
* 팀원 중에 이전에 관심이 있던 역사마스터 API를 이용한 아이디어로 시작
* 역사마스터 API = 서울시 지하철 역 또는 시내버스 정류장 주위 반경으로 여행지를 보여주는 API
* 여기에 혼잡도를 추가한 혼잡한 여행지를 고려할 때 비슷한 여행지로 우회추천하는 형식에 지도를 만들자
* 이를 kakao 지도맵 API와 연동해서 웹 완성을 목표로 함

### 데이터 센터 화재 사건 발발

* 프로젝트 진행중 데이터센터 화재로 서울시 실시간 도시데이터 사용 불가  

#### 트러블 슈팅

1. 사이트에 유동인구 7년치 학습데이터가 있는걸 전부 수집해 현재 시간을 예측하는 API 서버 생성
2. 정확한 위치 정보를 위해 집계구 단위를 범주형 변수로 지정
3. 이를 csv파일에 feature를 맞춰서 데이터 가공 후 parquet 포맷으로 변경
4. lgbm 모델이 이 csv 데이터를 전부 받아들이지 못해서 코로나기간을 제외한 나머지 기간의 데이터를 1년치로 나눔
5. 학습들을 1년치씩 나눈것을 다시 합한 후 유동인구 예측
6. 이를 SK 장소혼잡도 공식으로 계산 후 Flask서버로 배포하여 Spring boot 서버에서 받아서 그 현 위치에 혼잡도를 알려줌

## 2. 주요 기능

### 4.3 기여한 부분

### 1. AppShell.tsx — 최상위 레이아웃 래퍼

* **역할**: 앱 전체를 감싸는 max-width 컨테이너. 배경색과 최대 너비를 props로 받아 적용
* **구조**: 단일 `<div>`로, `width` (기본 1280px), `background` (기본 `#0e1117` 다크 배경)을 인라인 스타일로 설정
* **특징**: 서버 컴포넌트 (`"use client"` 없음). 모바일에서는 100% 너비, PC(`lg:`)에서는 중앙 정렬

### 2. RootShell.tsx — 클라이언트 레이아웃

* **역할**: AppShell 내부에서 AsideBar + main 콘텐츠 + MenuDrawer를 조합하고, **MenuDrawer의 open/close 상태**를 `useState`로 관리
* **구조**: `AppShell` > `flex` 컨테이너 > `AsideBar` + `<main>{children}</main>` + `MenuDrawer`
* **상태**: `menuOpen` (boolean) — AsideBar의 햄버거 버튼 클릭 시 `true`, MenuDrawer 닫힘 시 `false`

### 3. AsideBar.tsx — 좌측 네비게이션 바

* **역할**: 왼쪽 사이드바(데스크톱) 및 하단 네비게이션 바(모바일)
* **구조**:
  * **데스크톱** (`hidden md:block`): 폭 64px의 세로 바. 햄버거 메뉴 버튼, `/` (지도) 링크, `/CongestionMap` (혼잡도 지도) 링크 3개
  * **모바일** (`md:hidden`): 화면 하단 고정 nav. "메뉴 / 지도 / 맵 / 레이어" 4개 버튼 (backdrop-blur 배경)
* **활성 표시**: `usePathname()`으로 현재 경로를 읽어 해당 링크에 `bg-blue-600 text-white` 스타일 적용
* **Props**: `onMenuOpen` — 클릭 시 부모(RootShell)의 `setMenuOpen(true)` 호출

### 4. MenuDrawer.tsx — 슬라이드 메뉴

* **역할**: 왼쪽에서 슬라이드되는 사이드 메뉴. 로그인, 즐겨찾기, 최근 검색, 내 리뷰, 설정 등 메뉴 항목 제공
* **구조**: Headless UI의 `Transition.Root` + `Dialog` 조합
  * **Backdrop**: 반투명 검정 오버레이 (`bg-black/40`), fade 애니메이션
  * **Panel**: 좌측에서 슬라이드인 (`-translate-x-full` → `translate-x-0`), 모바일 82vw/PC 420px 폭
* **메뉴 섹션**: 헤더(인사말 + 로그인 버튼) → 섹션1(즐겨찾기/최근검색/내리뷰/오프라인지도) → 섹션2(앱공유/설정/도움말)
* **부수효과**: `open` 변경 시 `document.body`에 `menu-open` 클래스 토글 (모바일 AI 배지 숨김용)

## 메인 지도 페이지 (`/`) 컴포넌트

### 5. MapContainerClient.tsx — 메인 라우터

앱의 **가장 큰 종합 클라이언트 컴포넌트**. 모든 지도 관련 상태와 로직을 소유합니다.

**Props** (서버에서 주입):

* `seoulBoundary: SeoulPoly | null` — 서버에서 계산된 서울 행정 경계 폴리곤
* `categoryItems: CatItem[]` — 카테고리 필터 목록

**상태 (20개+ useState)**:

| 상태 그룹   | 변수들                                                                                                                                       | 용도                              |
| ----------- | -------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------- |
| 지도/위치   | `center`, `myLocation`                                                                                                                       | 지도 중심 좌표, 현재 위치         |
| 검색        | `query`, `isSearchFocused`, `suggestions`                                                                                                    | 검색어, 포커스 여부, 자동완성     |
| 결과        | `results`, `selectedPlace`, `resultsPanelOpen`                                                                                               | 검색 결과 배열, 선택 장소, 패널   |
| 카테고리    | `selectedCat`                                                                                                                                | 현재 선택된 카테고리 코드         |
| 길찾기      | `origin`, `destination`, `routePanelOpen`, `routeSegments`, `travelMode`, `sortOption`, `walkSummary`, `transitRoutes`, `activeTransitIndex` | 출발/도착지, 경로 데이터          |
| AI 채팅     | `aiChatOpen`, `aiQuery`, `messages`, `isLoading`                                                                                             | AI 패널 상태                      |

**주요 기능 함수들**:

1. **`handleKakaoSearch(q)`** — 카카오 키워드 검색 실행. `placesRef`로 검색 후 `turf.booleanPointInPolygon()`으로 서울 경계 내 결과만 필터링. 필터링된 결과에 `fetchCongestionData()`로 혼잡도 데이터를 병합하고 `results`에 저장

2. **`fetchCongestionData(places)`** — `/api/congestion`에 좌표 배열을 POST하여 각 장소의 혼잡도(여유/보통/약간붐빔/붐빔) 레벨을 받아 기존 `AppPlace[]`에 병합

3. **`handleRecommendSearch(q)`** — 검색어를 지오코딩(`geocodeQuery`)하여 좌표로 변환 후 `/api/recommend`에 위치/시간/카테고리를 쿼리. 백엔드에서 카테고리별 장소 추천 결과를 받아 `results`에 저장

4. **`handleGetDirections(start, end, mode, sort)`** — `/api/route-directions`에 출발/도착 좌표 + 모드(walk/transit) + 정렬 옵션을 POST. 응답에 따라:
   * **도보**: `congestionPoints` 배열을 `parseAndSetPolyline()`으로 색상별 폴리라인 세그먼트로 변환
   * **대중교통**: `transitRoutes` 배열을 저장, `buildTransitPolylineSegments()`로 첫 번째 경로의 폴리라인 생성

5. **`handleAiSummit(textOverride?)`** — `/api/chat`에 대화 내역을 POST하여 Gemini AI 응답을 받아 `messages`에 추가

6. **`handleGetCurrentLocation()`** — `navigator.geolocation.getCurrentPosition()`으로 현재 위치 탐지. 7초 타임아웃 시 서울 시청 좌표(`INITIAL_CENTER`)로 폴백

7. **`geocodeQuery(q)`** — 카카오 Geocoder로 주소 → 좌표 변환. 실패 시 키워드 검색으로 폴백

**렌더링 구조**:

```
<div> (전체 컨테이너, relative)
  ├─ <Map> (카카오 지도, absolute inset-0)
  │    ├─ MapMarker (내 위치 — 파란 마커)
  │    ├─ MapMarker[] (검색 결과 마커들 — 선택 시 active 스타일)
  │    ├─ MapMarker (출발지 — 초록 마커, 길찾기 시)
  │    ├─ MapMarker (도착지 — 빨간 마커, 길찾기 시)
  │    └─ Polyline[] (경로 폴리라인 — 혼잡도별 색상)
  │
  ├─ ResultPanel (검색 결과 슬라이드 패널)
  ├─ RoutePanel (길찾기 경로 패널)
  │
  ├─ <div> (상단 UI, absolute top-4)
  │    ├─ TopSearchBar + AutoComplete
  │    ├─ "추천" 버튼
  │    ├─ Categories (카테고리 탭)
  │    └─ AiButton
  │
  ├─ AiAssistantPanel (AI 채팅 오버레이)
  └─ CurrentLocationButton (우하단 위치 버튼)
```

**인포윈도우 (useEffect)**: 마커 클릭으로 `selectedPlace`가 설정되면 카카오 InfoWindow를 DOM으로 직접 생성. 장소명/주소/카테고리/전화/카카오맵 링크/혼잡도를 표시하고 "여기서 출발" / "여기로 도착" 버튼으로 `origin`/`destination` 설정 가능

### 6. TopSearchBar.tsx — 상단 검색바

* **역할**: 검색 입력 필드 + 자동완성 드롭다운 컨테이너
* **구조**: `<header>` > `<form>` (둥근 카드 형태, 검색 아이콘 + input + clear/spinner) + 드롭다운 영역
* **동작**:
  * 입력값 변경 → `onQueryChange` 콜백
  * 포커스 시 `isFocused=true` → 하단에 `children` (AutoComplete)이 드롭다운으로 나타남
  * Enter 또는 form submit → `onSearch` 콜백
  * Clear 버튼 → `onClear` 콜백으로 전체 초기화
  * `isLoading=true` 시 clear 버튼 대신 회전 스피너 표시
* **블러 처리**: `onBlur`에 120ms 지연을 두어 자동완성 클릭이 먹히도록 함

### 7. AutoComplete.tsx — 자동완성 목록

* **역할**: TopSearchBar 드롭다운 내부에 표시되는 검색 제안 목록
* **Props**: `suggestions: AutoCompleteItem[]` (id, place_name, road_address_name, x, y), `onSelect` 콜백
* **구조**: `<ul>` 리스트. 각 항목에 검색 아이콘(`lucide-react` Search) + 장소명 + 도로명 주소 표시
* **동작**: 항목 클릭 → `onSelect(item)` → MapContainerClient에서 `query`에 장소명 설정, 포커스 해제
* **자동완성 데이터 흐름**: MapContainerClient의 `useEffect`에서 `query` 변경 시 200ms 디바운스로 카카오 `keywordSearch` 호출 → 상위 10개 결과를 `suggestions` state에 저장 → AutoComplete에 전달

### 8. Categories.tsx — 카테고리 필터 탭

* **역할**: 관광지/문화시설/행사 등 8개 카테고리를 가로 스크롤 탭으로 표시
* **Props**: `items: CatItem[]` (code, name, icon), `value: string|null` (현재 선택 코드), `onChange` 콜백
* **구조**: 가로 스크롤 가능한 `<div>` 안에 버튼들. 선택된 항목은 `bg-blue-600 text-white` + X 닫기 아이콘 오버레이
* **동작**: 같은 카테고리 재클릭 또는 X 클릭 → `onChange(null)`로 해제. 다른 카테고리 클릭 → `onChange(code)`로 전환

### 9. ResultPanel.tsx — 검색 결과 패널

* **역할**: 검색 또는 추천 결과를 좌측 슬라이드 패널로 표시
* **Props**: `open`, `items: AppPlace[]`, `activeId`, `onSelect`, `onClose`, 레이아웃 props (panelWidth, topRem 등)
* **구조**:
  * **모바일**: 전체화면 오버레이 (상단 9rem부터), 뒤로가기 버튼 포함
  * **데스크톱**: 좌측 360px 고정 패널, 우측에 접기 핸들 버튼
* **각 항목 표시**: 장소명 + 혼잡도 배지 (있을 경우) + 주소 + 거리(km)
* **활성 표시**: 선택된 항목에 `border-emerald-500 bg-emerald-50` 스타일

### 10. RoutePanel.tsx — 길찾기 경로 패널

* **역할**: 출발지→도착지 경로를 상세히 표시하는 좌측 패널
* **구조**: `<aside>` (360px 고정 패널)
  * **헤더**: 뒤로가기 버튼 + "경로 안내" 제목
  * **출발/도착 표시**: 출/도 아이콘 + 장소명
  * **모드 토글**: "도보" / "대중교통" 버튼 (현재 모드에 `bg-blue-600 text-white`)
  * **본문**: 모드에 따라 `RenderWalkRoute` 또는 `RenderTransitRoutes` 렌더링
* **`RenderWalkRoute`** (도보):
  * 소요시간/거리/혼잡도 점수 요약
  * 정렬 옵션 select (최적시간순/혼잡도순)
  * turn-by-turn 안내 목록 (`instructions[]`)

* **`RenderTransitRoutes`** (대중교통):
  * 여러 후보 경로를 `TransitRouteCard`로 표시
  * 각 카드: 소요시간/노선 요약/총거리/도보거리/요금 표시
  * 클릭 시 펼쳐서 `SegmentDetail` 상세 표시 (타임라인 UI)

* **`SegmentDetail`**: 각 구간(도보/버스/지하철)을 타임라인 점 아이콘 + 모드명/노선/시작→끝/소요시간/거리/혼잡도 + 상세 step 리스트로 표시

### 11. AiButton.tsx — AI 어시스턴트 열기 버튼

* **역할**: 우상단에 표시되는 "AI 어시스턴트" FAB 버튼
* **구조**: 단일 `<button>`, 파란→보라 그라디언트 배경, rounded-full, 흰 텍스트
* **동작**: 클릭 → MapContainerClient의 `setAiChatOpen(true)`

### 12. AiAssistantPanel.tsx — AI 채팅 패널

* **역할**: Google Gemini 기반 여행 도우미 채팅 오버레이
* **구조**: Headless UI `Transition.Root` + `Dialog`
* **모바일**: 화면 하단에서 올라오는 패널, 96vw 너비, 60vh 높이, 상단 라운드
  * **PC**: 우측에서 슬라이드인, 430px 폭, 전체 높이
  * 배경 오버레이 없음 (주석 처리됨 — 지도를 보면서 대화 가능)
* **레이아웃**: flex-col 3단 구조
  1. **헤더**: 그라디언트 배경 (blue→purple), 타이틀 "AI 어시스턴트" + 닫기 버튼
  2. **대화 영역**: flex-1 스크롤. 메시지 배열을 순회하며:
     * `recommendation` 필드가 있으면 → 클릭 가능한 추천 질문 버튼 (그라디언트)
     * `text` 필드가 있으면 → 유저(오른쪽, 파란 배경) / AI(왼쪽, 연한 파란 배경) 말풍선. AI 응답은 `ReactMarkdown`으로 마크다운 렌더링
  3. **입력폼**: 하단 고정. `<input>` + 전송 버튼 (그라디언트)
* **자동 스크롤**: `useEffect`로 `messages` 변경 시 스크롤 컨테이너를 맨 아래로 이동

### 13. CurrentLocationButton.tsx — 현재 위치 버튼

* **역할**: 우하단에 표시되는 원형 FAB. 클릭 시 현재 위치로 지도 이동
* **구조**: 흰 배경 원형 버튼 (48px 모바일 / 56px PC), 십자선 SVG 아이콘
* **동작**: 클릭 → MapContainerClient의 `handleGetCurrentLocation()` 호출
